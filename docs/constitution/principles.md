# Principles

- Train decisions, not predictions.
- Reward discipline over profit.
- Simulation before theory.
- Replay before explanation.
- Visual learning before text.
- Build engines, not levels.
- Build systems, not features.
- The core must remain domain-agnostic.
- The platform must remain plugin-first.
- Competency matters more than completion.
- Zero-cost, solo-founder development is the default constraint.
- Open-source tooling is preferred whenever practical.
