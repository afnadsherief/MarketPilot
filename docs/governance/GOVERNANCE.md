# Governance

## Purpose
Define how MarketPilot documentation and architecture are controlled.

## Rules
- Ratified architecture is frozen unless amended.
- Every approved artifact must have a canonical place.
- No approved concept may disappear.
- Documentation must reflect implementation milestones.
- RepositoryPilot will eventually automate repository maintenance.

## Required Artifacts
- Project status
- Roadmap
- Changelog
- Canonical IDs
- Founder decisions
- Repository health
- Quality gates
