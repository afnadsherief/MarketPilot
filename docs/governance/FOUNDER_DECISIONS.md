# Founder Decisions

## Decision Log

### FD-001
- Decision: Use a domain-agnostic, plugin-first architecture.
- Reason: Preserve future flexibility beyond trading.

### FD-002
- Decision: Keep architecture frozen unless amended through ADR.
- Reason: Prevent architectural drift.

### FD-003
- Decision: Use GitHub as the canonical documentation source and Qwen volumes as compiled distributions.
- Reason: Preserve maintainability and long-term consistency.

### FD-004
- Decision: Add RepositoryPilot as a future documentation-maintenance subsystem.
- Reason: Automate repository governance over time.
